# migration

Staging area for the WordPress-to-Next.js migration. Everything here except this README is gitignored.

## Expected layout

```
migration/
├── README.md                                                      (committed)
├── source/
│   └── hunyamunyarecords.WordPress.2026-04-22.xml                 (gitignored; do not commit)
├── raw.json                                                       (generated by wp-import; gitignored)
├── report.md                                                      (generated by wp-import; gitignored)
├── dead-links.md                                                  (generated by wp-verify; gitignored)
└── media-manifest.json                                            (generated by media-rehost; gitignored by default)
```

## Scripts (run from `/site`)

| Command | Purpose |
|---|---|
| `npm run import:wp` | Parse the WP XML, emit MDX under `site/content/` |
| `npm run rehost:media` | Download WP uploads, transcode, push to R2, write `media-manifest.json` |
| `npm run build:redirects` | Generate `site/public/_redirects` from legacy slugs |
| `npm run verify:wp` | Crawl live WP via `WP_LEGACY_APP_PASSWORD`, diff against migrated content, produce `dead-links.md` |

See `docs/specs/migration-dry-run.md` and `docs/specs/rebuild-v1.md §8` for classification rules, open questions, and flagged items.
